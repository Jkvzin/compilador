💡 WHAT THIS DOES
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

The symbol table is crucial for:

1. TRACKING VARIABLES
   - Store what variables are declared
   - What type each has (int, float, etc.)
   - Where they were declared (line/column)

2. DETECTING ERRORS
   - Prevent double declarations
   - Catch use of undefined variables

3. SCOPE MANAGEMENT
   - Support nested scopes (foundation for procedures)
   - Clean up variables when scope exits
   - Find most recent in scope

4. CODE GENERATION
   - Know variable types for type checking
   - Know where to emit code for declarations
   - Support type casting/conversion
