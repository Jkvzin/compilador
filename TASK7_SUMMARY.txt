## Operator Precedence Implementation

```
┌─────────────────────────────────────────┐
│ Precedence (Lowest to Highest)          │
├─────────────────────────────────────────┤
│ 1. Comparison   =, <>, <, >, <=, >=    │ → parser_parse_comparison()
│ 2. Term        +, -                    │ → parser_parse_term()
│ 3. Factor      *, /                    │ → parser_parse_factor()
│ 4. Unary       -, nao                  │ → parser_parse_unary()
│ 5. Primary     numbers, ids, (expr)    │ → parser_parse_primary()
└─────────────────────────────────────────┘
```

Each function calls the next higher precedence level, naturally enforcing precedence rules.

## Example: How Parsing Works

### Example 1: Simple Expression
```
Input: 2 + 3
Parse Tree:
    BINOP
    /    \
  2(+)   3
```

### Example 2: Operator Precedence
```
Input: 2 + 3 * 4
Parse Tree:
      BINOP(+)
      /        \
    2          BINOP(*)
              /        \
            3          4

Evaluates as: 2 + (3 * 4) = 14 ✓
Not as: (2 + 3) * 4 = 20 ✗
```

### Example 3: Parenthesized Expression
```
Input: (2 + 3) * 4
Parse Tree:
        BINOP(*)
       /        \
    BINOP(+)    4
    /     \
   2      3

Evaluates as: (2 + 3) * 4 = 20 ✓
```
